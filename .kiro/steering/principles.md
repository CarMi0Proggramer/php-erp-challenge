# Principles

- I prioritize clarity over cleverness
- I write code for maintainability, not speed of writing
- I prefer explicit logic over magic
- I optimize for long-term scalability, not short-term hacks
- I treat code as a product, not just a solution

## Decision Making

- If something is unclear → make it explicit
- If logic is reused → extract it
- If something feels complex → simplify before adding features
